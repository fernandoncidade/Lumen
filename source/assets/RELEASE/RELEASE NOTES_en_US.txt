Detailed description of main changes

1) Drag & Drop (DragDropTaskList)
- New custom QListWidget implementation for drag & drop with refined control:
  - Drag enabled, drops accepted, visible drop indicator and configured drop mode.
  - Forced SelectionMode SingleSelection to prevent accidental multi-item drags caused by residual selections.
  - mousePressEvent adjusted to ensure only the clicked item is selected/dragged (setCurrentItem / clearSelection / setSelected).
  - dragEnterEvent and dragMoveEvent accept the proposed action, improving interoperability between lists.
  - dropEvent rewritten to:
    - Correctly handle drops coming from the same list (fall back to default behavior and persist).
    - Validate that the source is a QListWidget and reject invalid drops.
    - Iterate only over selected items (single-selection guarantee) and call app.move_item_between_lists for consistent movement preserving metadata.
    - Block signals from the lists during the operation to avoid callback side effects.
    - Clear selection on the source after the operation to prevent additional removals by Qt's internal mechanism.
    - Force setDropAction(Qt.CopyAction) + event.accept() at the end of the drop to prevent Qt from performing an automatic removal (MoveAction) that caused deletion of remaining items in the source quadrant.
  - Robust error logging in dropEvent to aid diagnostics.

Impact of the fix
- Fixes the issue where dragging a task would remove all other tasks from the source quadrant (double removal caused by Qt's final MoveAction). The chosen strategy ensures the application explicitly controls removal and prevents Qt from finalizing the move by inadvertently removing remaining items.

2) CustomTimeEdit
- Implemented CustomTimeEdit (subclass of QTimeEdit) with:
  - Wrapping enabled and stepBy adapted for continuous minute increments with wrap-around 0–1439.
  - Robust behavior when navigating time sections (minutes vs other sections).
  - Internal storage of the last time value for potential future uses.

3) Locale and Date/Time Format
- Applied locale to QDateEdit based on the translation manager, with a safe fallback:
  - Explicit support for pt-BR and en-US when available.
  - Determination of displayFormat derived from the locale; regex adjustments to ensure a 4-digit year.
  - Connection to the idioma_alterado (language_changed) signal to reapply locale dynamically.
- time_input (CustomTimeEdit) keeps displayFormat "HH:mm" with a handler to reapply locale / update the time-checkbox text when the language changes.

4) Placeholders and UI Integration
- Quadrant lists and completed-lists are instantiated as DragDropTaskList and receive initial placeholders via app.add_placeholder.
- itemChanged connections established to synchronize check/uncheck with automatic moves (app.handle_item_checked).
- Context menu: each list registers customContextMenuRequested connected to app.show_context_menu.

5) Palette / Theme Synchronization
- Utility functions to apply the window background color as Base/AlternateBase across Matrix widgets:
  - _apply_window_as_base applies the palette to widgets and viewports (QListWidget.viewport when applicable).
  - Installation of an eventFilter (_PaletteThemeSyncFilter) to listen for ApplicationPaletteChange / PaletteChange / StyleChange / ThemeChange / ColorSchemeChange (when available) and reapply backgrounds automatically.
  - Exception handling with debug logging to avoid silent failures in restricted environments.

6) Robustness and Logging
- Consistent use of LogManager to record failures and exceptions at critical points (drag/drop, locale, signal connections).
- Blocking signals during sensitive operations (drop, list manipulation) to reduce callback side effects.

  - Pause behavior fixed:
    - Before: when the user paused playback, newly generated audio files (chunks) could start automatically, causing the reader to skip the pause.
    - Now: the _play_generated_audio function explicitly checks the player state (QMediaPlayer.PausedState) and the internal _is_paused flag. If the player is paused, new chunks do NOT start automatically — they are only added to the queue (_generated_queue) for later playback.
  - Additional protections:
    - Preservation of chunks during voice change: if _changing_voice is active, generated chunks are safely discarded (attempt to remove the file) and ignored.
    - Duplicate control: generated file paths are recorded in _generated_files to avoid duplicate reprocessing.
    - Playback state check: does not attempt to start new audio if already in PlayingState; respects PausedState.
  - Robustness and logging:
    - Expanded exception handling to avoid silent failures and ensure leitura_finalizada() is called when appropriate.
    - Log messages document errors when starting chunk playback and failures when cleaning up generated files.

Impact and expected behavior
- User pauses narration with the "⏸️ Pause" button: playback stops; newly generated chunks from the TTS mechanism are queued and do NOT start playing until the user manually resumes (▶️ Continue) or the player is in a state that allows automatic playback.
- Voice change during generation: chunks created during a voice change are safely discarded to avoid playback with the old voice.

How to test (quick steps)

3. While playing, click "⏸️ Pause".
4. Confirm playback stops and that newly generated chunks (e.g., for long texts) do not start automatically.
